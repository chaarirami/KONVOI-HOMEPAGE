  <how-to-verify>
Before DNS cutover, complete these steps IN ORDER:

**A. Fill in Rybbit domain (if self-hosted instance is ready)**
1. Open `src/components/common/Analytics.astro`
2. Replace `RYBBIT_DOMAIN` with your actual Rybbit domain (e.g. `analytics.konvoi.eu`)
3. Replace `RYBBIT_SITE_ID` with the site ID from your Rybbit dashboard
4. Open `public/_headers`
5. Replace both occurrences of `RYBBIT_DOMAIN` with the same actual domain
6. Run `pnpm build` to verify build still passes
7. Commit: `git add -A && git commit -m "feat(07): set Rybbit analytics domain"`

   If Rybbit is NOT yet provisioned: skip this step. The placeholder is harmless — the script will 404 silently. You can update after launch.

**B. Connect Netlify custom domain**
1. Go to Netlify Dashboard → your site → Site configuration → Domain management
2. Click "Add a domain alias" → enter `konvoi.eu`
3. Click "Add a domain alias" again → enter `www.konvoi.eu`
4. Netlify will show you the DNS records to set:
   - `A` record for `@` (apex) → Netlify load balancer IP (shown in dashboard)
   - `CNAME` record for `www` → your-site-name.netlify.app

**C. Update DNS at your registrar**
1. Log in to your domain registrar (where konvoi.eu is registered)
2. Remove existing Jimdo DNS records (A, CNAME entries pointing to Jimdo)
3. Add the Netlify DNS records from step B
4. Set TTL to 300 seconds (5 min) for fast cutover
5. Save. Propagation typically takes 5–60 minutes.

**D. Verify cutover**
Run from your terminal:
```bash
# Check DNS resolution (repeat until it shows Netlify IP)
nslookup konvoi.eu

# Check the live site
curl -I https://www.konvoi.eu

# Verify hreflang in live HTML
curl -s https://www.konvoi.eu | grep hreflang

# Verify sitemap is accessible
curl -I https://www.konvoi.eu/sitemap-index.xml

# Verify _redirects works (should 301 to /produkt/)
curl -I https://www.konvoi.eu/die-loesung
```

Expected results:
- `curl -I https://www.konvoi.eu` returns HTTP 200 with `content-type: text/html`
- `curl -s https://www.konvoi.eu | grep hreflang` returns 3 hreflang link tags
- `curl -I https://www.konvoi.eu/die-loesung` returns `HTTP/2 301` with `location: /produkt/`
- Netlify dashboard shows SSL certificate issued for konvoi.eu (green lock)

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Search Console International Targeting and netlify.app noindex verification</name>
  <what-claude-automated>
The live site is now on Netlify. The sitemap with hreflang alternates is at https://www.konvoi.eu/sitemap-index.xml. The Metadata.astro component emits hreflang link tags on every page. Layout.astro emits noindex on deploy-preview and branch-deploy contexts.
  </what-claude-automated>
  <how-to-verify>
Complete these Search Console steps after DNS has propagated:

**A. Add and verify Search Console properties**
1. Go to https://search.google.com/search-console
2. Add property: `https://www.konvoi.eu` (URL-prefix property)
3. Verify ownership — use the HTML file method (download file → place in public/ → deploy → verify) OR use the DNS TXT record method
4. After verification, go to Settings → International Targeting
5. Set country targeting to "Germany" for the DE property
6. Repeat: Add property `https://www.konvoi.eu/en/` (the EN hreflang root)
7. Submit sitemap: In each property, go to Sitemaps → enter `sitemap-index.xml` → Submit

**B. Check for hreflang errors**
1. In Search Console, go to Enhancements → International Targeting
2. After a few days of crawling, check for "Return tag missing" errors
3. If hreflang errors appear, verify that the DE page has an hreflang pointing to EN AND the EN page has an hreflang pointing to DE (reciprocal requirement)

**C. Verify netlify.app subdomain is NOT indexed (DEPLOY-05)**
1. In a private browser window, search Google for:
   ```
   site:netlify.app konvoi
   ```
2. Expected result: Zero results, OR only the your-site-name.netlify.app URL
3. If netlify.app pages ARE indexed:
   - Add `X-Robots-Tag: noindex` to `public/_headers` for the netlify.app domain:
     ```
     # In public/_headers, add before the /* rule:
     # Note: Netlify _headers cannot scope by hostname. Use Google Search Console
     # URL removal tool to request removal of specific netlify.app URLs.
     ```
   - Use Search Console URL removal tool to remove any netlify.app URLs
   - The long-term fix: Netlify automatically adds `X-Robots-Tag: noindex` to
     all *.netlify.app traffic when a custom domain is the primary domain —
     verify this is active in Netlify Dashboard → Domain management

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DNS registrar→Netlify | DNS propagation moves traffic from Jimdo to Netlify |
| browser→SSL cert | Netlify auto-provisions Let's Encrypt cert for custom domain |
| Google crawler→sitemap | Submitting sitemap exposes all page URLs to Google |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05-01 | Denial of Service | DNS propagation gap (brief period when neither Jimdo nor Netlify serves) | accept | Keep Jimdo DNS records alive for 24h overlap; reduce TTL to 5min before cutover |
| T-07-05-02 | Spoofing | netlify.app subdomain indexed before custom domain takes over | mitigate | Verify site:netlify.app returns zero; use Search Console removal if needed; Netlify auto-noindexes *.netlify.app when custom domain is primary |
| T-07-05-03 | Info Disclosure | Search Console exposing all indexed URLs | accept | Marketing site — all pages intentionally public; Search Console access restricted to authorized Google account |
</threat_model>
