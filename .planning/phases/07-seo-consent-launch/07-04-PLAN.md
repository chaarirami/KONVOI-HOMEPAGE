**Step 1: Rewrite public/_headers**

Replace the current minimal content with the full CSP + security headers block:

```
# public/_headers — Netlify header rules
# Per CONTEXT.md D-15: allow Web3Forms, Turnstile, Google Maps, Rybbit, self-hosted fonts
# Per RESEARCH.md Pattern 9: CSP directives verified against official sources
# BEFORE LAUNCH: Replace RYBBIT_DOMAIN with actual self-hosted Rybbit domain

/*
  Content-Security-Policy: default-src 'self'; script-src 'self' 'unsafe-inline' https://challenges.cloudflare.com https://maps.googleapis.com https://RYBBIT_DOMAIN; style-src 'self' 'unsafe-inline'; img-src 'self' data: https://maps.gstatic.com https://*.googleapis.com https://*.gstatic.com; font-src 'self'; connect-src 'self' https://api.web3forms.com https://RYBBIT_DOMAIN; frame-src https://challenges.cloudflare.com https://www.google.com; object-src 'none'; base-uri 'self'; form-action 'self' https://api.web3forms.com;
  X-Frame-Options: DENY
  X-Content-Type-Options: nosniff
  Referrer-Policy: strict-origin-when-cross-origin
  Permissions-Policy: geolocation=(), microphone=(), camera=()

/_astro/*
  Cache-Control: public, max-age=31536000, immutable
```

Notes on CSP choices (per RESEARCH.md):
- `'unsafe-inline'` in `script-src`: Required for Astro inline scripts (ApplyColorMode, BasicScripts) and vanilla-cookieconsent initialization. For a static marketing site, this is the pragmatic choice. Do NOT add `'unsafe-eval'`.
- `'unsafe-inline'` in `style-src`: Required for vanilla-cookieconsent v3 which injects inline styles for the banner.
- `https://RYBBIT_DOMAIN`: Placeholder — user must replace with actual self-hosted domain before DNS cutover. Appears in both `script-src` and `connect-src`.
- `https://maps.googleapis.com`, `https://www.google.com`, `https://maps.gstatic.com`, `https://*.googleapis.com`: Required for Google Maps (consent-gated functional category). Must be present in CSP even though Maps only loads post-consent.
- `form-action`: Restricts where forms can POST — only to self and Web3Forms API endpoint.
- No `https://cal.eu` needed: Cal.eu is only an external `<a href>` link, not an iframe embed (per RESEARCH.md Assumption A1).
- No Formspree, Plausible, or Jimdo entries (per CONTEXT.md D-15).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser→Netlify | All HTTP responses governed by _headers rules |
| form POST→api.web3forms.com | Only allowed form-action target besides 'self' |
| browser→Google Maps | Only loads post functional-cookie consent; CSP allows it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-04-01 | Tampering | Clickjacking via iframe embedding | mitigate | X-Frame-Options: DENY blocks all iframe embedding of konvoi.eu pages |
| T-07-04-02 | Info Disclosure | MIME sniffing attacks | mitigate | X-Content-Type-Options: nosniff prevents MIME-type sniffing |
| T-07-04-03 | Info Disclosure | Referrer leakage on external navigation | mitigate | Referrer-Policy: strict-origin-when-cross-origin sends origin only, not full path |
| T-07-04-04 | Tampering | XSS via inline script injection | mitigate | CSP script-src blocks eval; 'unsafe-inline' required for Astro inline scripts — acceptable for static marketing site threat model |
| T-07-04-05 | Tampering | Unauthorized form POST targets | mitigate | form-action CSP directive restricts POST targets to 'self' and api.web3forms.com only |
| T-07-04-06 | Elevation of Privilege | Device sensor API abuse (geolocation, camera) | mitigate | Permissions-Policy disables geolocation, microphone, camera |
| T-07-04-07 | Denial of Service | Redirect loops from catch-all redirect | accept | Catch-all commented out; specific rules only; Netlify 404 page serves unknown paths |
</threat_model>
