The component must:
1. Import `roiFormulas` from `~/data/pricing` (not hardcode values)

3. Accept props: `{ locale: 'de' | 'en'; embedMode?: 'compact' | 'full' }`
4. Manage state: `fleetSize` (number, default 10), `vertical` (string, default 'high_value'), `frequency` (number, default 5), `result` (null or computed object)
5. Implement `calculateRoi()` using roiFormulas — exact formula:

```typescript
const formula = roiFormulas.byVertical[vertical];
const annualTheftCost = fleetSize * formula.annualTheftCostPerVehicle;
const konvoiSavings = annualTheftCost * formula.savingsRate;
const annualKonvoiCost = roiFormulas.monthlyPriceEur * fleetSize * 12;
// De-minimis: 80% of annual Konvoi cost, capped per vehicle and per company
const rawSubsidy = annualKonvoiCost * roiFormulas.deMinimisRate;
const perVehicleCap = fleetSize * roiFormulas.deMinimisMaxPerVehicle;
const deMinimisReimbursement = Math.min(rawSubsidy, perVehicleCap, roiFormulas.deMinimisMaxPerCompany);
// Payback: months until savings cover annual Konvoi cost
const monthlySavings = konvoiSavings / 12;
const paybackMonths = monthlySavings > 0 ? Math.round(annualKonvoiCost / monthlySavings) : 0;

setResult({
  fleetSize,
  vertical,
  annualTheftCost: Math.round(annualTheftCost),
  konvoiSavings: Math.round(konvoiSavings),
  deMinimisReimbursement: Math.round(deMinimisReimbursement),
  paybackMonths,
});
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser user input → ROI formula | Fleet size, vertical, frequency are user-provided numbers processed client-side only |
| ROI result → URL query params | Results written to URL as query params for ConsultForm pre-fill |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Injection | RoiCalculator inputs | mitigate | Parse fleet_size/frequency with `parseInt(val, 10)` and clamp with `Math.max(1, Math.min(1000, val))` before formula use; never render raw user input as HTML |
| T-05-02-02 | Tampering | URL pre-fill params | accept | Params are fleet_size, vertical, estimated_savings — no auth tokens, no secrets; ConsultForm reads and displays them as pre-filled values only; Formspree validates on submission |
| T-05-02-03 | Information Disclosure | roiFormulas in client bundle | accept | Formula assumptions are intentionally public (calculator is a marketing tool); no PII or secrets |
| T-05-02-04 | Brute Force | ROI URL param manipulation | accept | Pre-fill params are non-sensitive display values; no server state changes from param manipulation |
</threat_model>
