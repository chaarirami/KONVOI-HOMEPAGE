<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry → build | @fontsource packages fetched from npm registry during pnpm install; consumed at build time only |
| CSS cascade → browser | CSS custom properties in CustomStyles.astro are public; no secrets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | @fontsource npm packages | mitigate | Versions pinned in pnpm-lock.yaml; use `pnpm install --frozen-lockfile` in CI (already enforced by Phase 1 workflow). Integrity guaranteed by lockfile hash. |
| T-02-02 | Disclosure | src/components/CustomStyles.astro | accept | CSS custom properties (colours, font names) are fully public by design — rendered in every HTML page's `<style>` tag. No sensitive data. |
| T-02-03 | Tampering | tailwind.css @custom-variant dark | mitigate | Pinned with LOAD-BEARING comment block in this plan (Task 1). Future editors cannot silently remove it without noticing the comment. |
| T-02-04 | Denial of Service | Font file size (~160KB) | accept | 120KB Montserrat + 40KB PT Serif is within standard marketing-site budget. Files served as static assets from Netlify CDN. No runtime risk. |
</threat_model>
